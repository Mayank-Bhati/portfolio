  • Edu-Profile: Designed batch read/write/update APIs with parallel processing for high-throughput profile operations
  • Reliability: Introduced DLQ and retry patterns for Hoolihan consumers; improved message durability and reduced data-loss incidents

Manch — AI-Powered Live Streaming Platform

Personal

Node.js · Express · PostgreSQL · MongoDB · Redis · FFmpeg · SRT/RTMP · HLS · SwiftUI (iOS)

AI-driven live streaming platform that automates multi-camera switching; replaces manual production with intelligent, real-time camera selection.

Problem: Traditional multi-camera streaming needs skilled operators, increases complexity, and limits scalability for smaller creators (sports, weddings, interviews).

Solution: Manch ingests multiple camera feeds, uses an AI decision engine to select the best view in real time, and outputs a single optimized stream. Supports event templates (sports, interviews) for customizable switching logic.

Architecture:

  • Ingestion — Accepts multiple RTMP/SRT camera streams
  • Processing — FFmpeg and media services to normalize and buffer streams
  • AI Decision Engine — Selects best camera based on motion, audio, and templates
  • Delivery — HLS/DASH via CDN for scalable playback
  • Control Plane — Dashboard for templates, streams, and configs

Mobile (manch-ios): SwiftUI + AVFoundation app using HaishinKit (RTMP); one-tap streaming, connects to backend ingest pipeline as a mobile camera node.

Features: Multi-camera ingestion · AI-based auto switching · Low latency (~2–5s) · Cloud architecture · iOS broadcasting · Template-driven logic (sports, events)

Technical: Microservices with clear separation of ingest, processing, AI, and delivery; CAP trade-offs for latency vs availability; integrated real-time media with AI inference pipelines.
