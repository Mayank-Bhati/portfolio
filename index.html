CareerCompass
FOUNDER · LIVE

An AI-assisted career-guidance platform that helps students discover study paths that fit who they are — then compare the real institutions behind them.

The problem

Italy has one of Europe's highest dropout rates, largely because students pick courses blindly — with no way to match their interests to paths, or to see the reality of a course before enrolling.

Five-stage flow
Express→ Reveal→ Discover→ Filter→ Compare
How it works
  • An adaptive mixed-format survey (binary, multi-select, Likert) maps interests onto a RIASEC psychometric model — different every run via stratified randomization
  • Deterministic, fully client-side scoring engines — cosine-similarity career matching plus a two-layer path/environment fit — run as pure functions
  • No backend, zero data stored: privacy-clean for minors by design, free to host, fully auditable — deployed static on GitHub Pages via CI/CD
  • Grounded in public education datasets (AlmaLaurea, MUR) for real employment, satisfaction, and dropout figures

Manch — AI Live-Streaming Platform
PERSONAL · SYSTEMS

An AI-driven multi-camera live-streaming platform that automates the job of a live production crew — ingesting several camera feeds and intelligently cutting between them in real time.

The problem

Multi-camera streaming — sports, weddings, interviews, conferences — needs a skilled human operator switching feeds live. That's expensive, doesn't scale, and puts good production out of reach for small creators.

Real-time media pipeline
Ingest · RTMP/SRT→ Normalize · FFmpeg→ AI Switcher→ Encode · HLS→ CDN
Engineering detail
  • Ingestion layer accepts multiple simultaneous RTMP/SRT camera feeds, each treated as an independent stream node the system can subscribe to
  • AI decision engine scores every feed in real time on motion, audio energy, and scene templates (sports vs. interview logic), then picks the "best" camera — the core replacement for a human switcher
  • Processing services use FFmpeg to normalize, buffer, and time-align feeds so switches are seamless despite per-camera latency and jitter
  • Delivery via HLS/DASH over CDN for scalable playback at ~2–5s latency, decoupling viewers from ingest load
  • Deliberate microservice separation of ingest / processing / AI / delivery, with explicit CAP trade-offs favoring latency over strict consistency on the live path
  • manch-ios: a SwiftUI + AVFoundation companion app using HaishinKit for one-tap RTMP broadcasting — turns any phone into a mobile camera node
